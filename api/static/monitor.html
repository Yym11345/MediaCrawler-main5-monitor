采集前准备

  1. 关闭 VPN、代理、网络加速器。保持运行服务的服务器网络直连目标平台。
  2. 准备可正常登录的平台账号；建议使用爬虫专用小号，不要使用矩阵大号扫码。
  3. 登录后不要立刻关闭浏览器，等待日志显示采集完成并保存快照。
  4. 新账号建议先用 50 到 300 条试跑，确认稳定后再提高；全量采集更慢，也更容易触发平台校验。

登录状态和浏览器环境

  1. 系统会为抖音、小红书、B站分别保存独立浏览器环境，首次扫码后下次会优先复用登录状态。
  2. 采集时不要开启 VPN、代理或网络加速器，否则保存的登录环境也可能被平台判定异常。
  3. 如果某个平台一直提示登录过期、账号切换不干净，先停止采集和网站服务，再删除对应平台的浏览器目录后重新扫码。
  4. 浏览器登录目录保存在运行服务的机器 browser_data 下，包含账号登录态，不要上传 GitHub，也不要发给别人。

监控账号采集

  1. 在“账号管理”中选择平台，只粘贴账号主页链接；不再支持直接输入 sec_uid、用户 ID 或 B站 UID。
  2. 点击“添加”，账号会进入监控列表；备注名称可选，便于区分矩阵账号。
  3. 点击账号行内的“采集”，只采集这个账号；点击顶部“立即采集”，采集当前平台已启用的账号。
  4. 打开的浏览器里完成扫码、滑块或手机确认后，等待状态从采集中进入收尾和保存快照。

单个作品采集

  1. 进入“数据看板”，在“单个作品”区域选择平台。
  2. 只粘贴作品链接或笔记链接；不再支持手动输入作品 ID。
  3. 需要二级评论时勾选“二级评论”，否则只采集一级评论。
  4. 单个作品导出会保留评论内容；监控账号和作品指标导出不包含评论内容。

数据和导出说明

  1. 数据看板中的粉丝、获赞和 24 小时变化来自账号快照；作品数来自当前数据库累计。
  2. “作品互动量”来自已采集作品表，口径是点赞数 + 评论数 + 分享数。
  3. “今日爆款作品 Top 10”按作品互动量从高到低排序，帮助快速查看当前已采集作品中的高互动内容。
  4. 账号 CSV 导出包含账号快照和作品指标：点赞量、评论量、分享、发布时间、链接。
  5. 账号 CSV 和监控面板作品“导出指标”不会导出评论内容。
  6. 历史快照是历史记录，重新采集后会生成新快照，不会自动改写旧快照。

常见问题

  1. 添加账号失败：请确认输入的是完整账号主页链接，并且平台选择和链接来源一致。
  2. 单作品采集失败：请确认输入的是完整作品链接或笔记链接，不要只填作品 ID。
  3. 登录过期或 412：关闭 VPN/代理，重新扫码登录后再采集。
  4. 作品数量不完整：平台分页接口可能触发风控，建议分批采集并减少单次数量。
  5. 粉丝数和主页显示不一致：重新采集该账号生成新快照；平台展示值通常会四舍五入。
  6. 按钮暂时不可点：通常是采集进程仍在运行、收尾或保存快照，等待日志结束后再操作。

推荐采集习惯

建议每 2 到 4 小时巡检一次矩阵账号。同一平台不要频繁切换账号或连续高频采集。账号监控主要用于老板 KPI、平台占比、作品指标和快照趋势；评论内容建议只在“单个作品采集”中按需获取，避免日志过慢和平台校验频繁出现。
